# Bitget AI Base Camp Hackathon S1: A Championship Strategy for a Solo Claude Code Builder

## TL;DR
- **Build for Track 2 (Trading Infra), not Track 1.** The single highest-probability cross-track winner is a verifiable agent-evaluation / paper-trading-and-leaderboard harness built directly on the Bitget Agent Hub. It is a reusable open-source tool that other hackathon contestants and Bitget's own developer community adopt, which compounds the exact "verifiable usage evidence" (API call volume + user count + sim trade logs) that every track is judged on. This converts your African / emerging-market distribution moat into a live, judge-visible metric no other solo builder can fake in four weeks.
- **The toolkit's biggest gap is your opening.** The Agent Hub (BitgetLimited/agent_hub, MIT, ~209 stars) gives you perception (5 analyst Skills) and execution (58 tools across 9 modules) but ships no backtesting engine, no paper-trading sandbox, no evaluation/scoring layer, and no guardrail/risk framework. Bitget's own Playbook (natural-language to backtest to deploy) is closed/beta. Build the missing open "trust layer" and you fill a real pain point rather than reproducing GetAgent/GetClaw.
- **Win the demo, not the idea.** Judges (led by Bitget CEO Gracy Chen, plus ~30 named judges) reward a flawless runnable demo with real evidence and a credible "this outlives the hackathon" story. A solo builder beats teams by being ruthlessly scoped, leak-free in backtests, and showing real users from Nigerian/Kenyan/SA Telegram groups generating real sim-trade telemetry on a public leaderboard.

### 2. The Bitget toolkit: what it really is, and where the gaps are
**Bitget Agent Hub** (github.com/BitgetLimited/agent_hub, redirects to Bitget-AI/agent_hub; MIT license; TypeScript 66% / Python 20% / JS 14%; ~209 stars, 15 forks, 69 commits) is a pnpm monorepo with three installable npm packages:
- **`bitget-client` (CLI `bgc`)**: shell access to all tools, JSON output.
- **MCP server (`bitget-mcp-server`)**: one-line `claude mcp add ... npx -y bitget-mcp-server`. Default loads spot+futures+account = **36 tools** (fits Cursor's 40-tool cap); `--modules all` loads the full **58 tools across 9 modules**: spot (13), futures (14), account (8), margin (7), copytrading (5), convert (3), earn (3), p2p (2), broker (3).
- **`bitget-skill-hub`**: 5 analyst Skills. `macro-analyst` (Fed/yield curve, BTC vs DXY/Nasdaq/Gold, RISK-ON/OFF verdict), `market-intel` (ETF flows, whales, DeFi TVL, cycle indicators like AHR999/Pi Cycle), `news-briefing` (20+ RSS feeds, even Weibo/Douyin pulse), `sentiment-analyst` (Fear & Greed, long/short, funding, OI), `technical-analysis` (23 indicators in 6 categories, local Python pandas/numpy, exports CSV for downstream backtesting).

Security model: env-var credentials only, `--read-only` flag, HMAC-SHA256, client-side rate-limit, write-ops require explicit confirmation. Most Skills depend on an external market-data MCP server (datahub.noxiaohao.com/mcp); `technical-analysis` instead hits the Bitget API directly and computes locally.

**Bitget Playbook / Bitget AI**: the natural-language to AI-generated strategy to backtest (PnL/max drawdown/Sharpe/win rate) to deploy-live pipeline. It connects via a `getagent` skill (npm `@bitget-ai/getagent-skill`) and **Getclaw** (Telegram @getclaw_official_bot), which is OpenClaw-based, runs at ~25-30% the cost of a standard OpenClaw config, and supports backtesting prompts ("calculate total return and max drawdown using the past six months on 15-minute timeframe, starting capital 10,000 USDT"). As of the Bitget AI rollout, the suite reports **1 million+ users and $1.2B in agent trading volume across 58 tools**; GetAgent alone reached ~450,000 users (Messari Pulse, April 2026). Playbook is in **beta** and is a closed marketplace.

**The exploitable gaps for a builder:**
- **No rigorous, leak-free backtesting/evaluation harness is open-sourced.** The Skills produce perception; nothing scores decision quality, prevents look-ahead bias, or decomposes returns into market/style/skill.
- **No paper-trading sandbox** that records auditable sim-trade logs ships in the repo (Track 2 explicitly invites one).
- **No guardrail/risk-management framework** beyond `--read-only` and confirmation prompts, no position limits, circuit breakers, drawdown kill-switches, or runtime policy enforcement.
- **No tokenized-stock data Skill**, the 5 Skills are crypto-only; nothing wraps Bitget Stocks 2.0 / Reality rTokens for Track 3.
- **Docs are thin** (no published releases, 5 open issues): an integration-friendly wrapper with great DX is itself a differentiator.

### 3. The competitive landscape in 2026 (what already exists, what's unsolved)
- **Autonomous agent frameworks:** ElizaOS (formerly ai16z) and Virtuals dominate; ElizaOS v2 added multi-agent architecture plus a "Global Trust Marketplace" for reputation/verifiable track records. Known weaknesses: complex codebases, weak planning, limited verifiable performance.
- **Live verifiable benchmarks are the cultural touchstone:** Nof1's **Alpha Arena** gave six LLMs (DeepSeek V3.1, GPT-5, Claude Sonnet 4.5, Gemini 2.5 Pro, Grok 4, Qwen3 Max) $10K each to trade Hyperliquid perps live, on-chain, with fully auditable wallets and a public leaderboard. In Season 1 (Oct 18 to Nov 3, 2025), **Qwen3 Max won with roughly +22% (about $12,287 from $10,000), DeepSeek V3.1 finished modestly positive (~+4-5%), while GPT-5 lost over 60% of starting capital and Gemini 2.5 Pro also suffered a major drawdown** (per iWeaver AI, with results independently confirmed by Yahoo Finance/Barron's/SCMP). This proved the format judges find compelling: identical rules, transparent logs, on-chain verification. RockAlpha extends the same concept to US equities.
- **The genuinely unsolved hard problems (your wedge):** (a) **evaluation rigor**, academic work (LiveTradeBench, PredictionMarketBench, Look-Ahead-Bench, the "From Knowing to Doing" memory-controlled benchmark) shows backtests routinely suffer information leakage, overfitting, and conflate market drift with skill; (b) **risk guardrails**, AgentSpec (ICSE'26) and circuit-breaker patterns are emerging but unadopted in crypto agents; (c) **verifiable, tamper-proof performance**; (d) the **Bank of England's Jonathan Hall**, in his May 7, 2024 speech "Monsters in the deep?" at the University of Exeter, warned of an emerging "monoculture" and said deep trading algorithms will need to be tested in multi-agent sandbox environments and constrained by tightly monitored risk and stop-loss limits. Safety is a live, credible, citable narrative.
- **Adjacent open tools to learn from / not duplicate:** **virattt/ai-hedge-fund** (~50,000+ GitHub stars as of early 2026, 51,700 stars and 9,000+ forks per Edgen, Jan 2026; React/TypeScript front end plus Python/FastAPI backend on LangGraph, 18 agents = 12 investor personas + 6 specialists, with a built-in backtesting module, sim-only), HKUDS/AI-Trader (agent-native, Polymarket paper trading), Hummingbot (MCP + skills), OpenAlgo (sandbox engine + PnL/latency dashboards + Telegram alerts), Open-Finance-Lab/AgenticTrading (Alpaca paper + backtest CLI).

### 6. Technical building blocks and best practices (late 2025/2026)
- **Backtesting rigor:** enforce point-in-time data, walk-forward validation, leakage audits/entity anonymization, and return decomposition (market vs style vs stock-specific skill). Report Sharpe, max drawdown, win rate, expectancy, and explicitly model fees/slippage (PredictionMarketBench shows naive LLM agents bleed on settlement/fees).
- **Architecture pattern:** the ReAct loop (observe to reason to decide to execute to remember) on frameworks like LangChain/CrewAI; specialized role agents (alpha, risk manager, execution) with a conductor; persistent memory (regime labels, rationales).
- **Risk guardrails in code, not prompts:** position caps, leverage limits, daily-loss circuit breakers (three-state closed/open/half-open), kill-switch, runtime policy enforcement (AgentSpec-style rules), structured JSON decision output for auditability.
- **Verifiable performance:** public leaderboard plus immutable trade journal; consider Alpha-Arena-style transparency (timestamped logs, reproducible runs). VPS heartbeat monitoring across all failure layers (process, broker conn, LLM conn, data feed, state file).
- **Stack fit:** your Next.js 15 / Vercel AI SDK / TS stack is ideal for the dashboard plus leaderboard front end; Bitget MCP plugs straight into Claude Code; the `technical-analysis` Skill's Python/CSV export feeds a backtester; Tencent Cloud VPS runs the 24/7 agent loop and cron jobs.

### 7. The "verifiable usage evidence" angle (the decisive criterion)
All tracks require at least one of: real/sim trade logs, API call volume, or user count. For a solo builder in ~4 weeks, the most credible and impressive evidence stack is, in priority order: (1) a **public leaderboard with auditable sim-trade logs** generated on real Bitget market data (Alpha-Arena credibility, zero capital risk, explicitly accepted); (2) **real users from your Telegram communities** producing genuine user counts and session telemetry; (3) **API-call telemetry** from your tool's own usage (especially if Track 2 devs adopt it). Stacking all three is far beyond what a typical contestant produces and directly maxes the baseline criterion.

## Details: Five concrete project concepts

**Concept A: "BitgetBench" (Track 2, recommended core).** An open-source, MIT, npm-installable evaluation plus paper-trading harness for Bitget Agent Hub agents. One command spins up a leak-free backtester (point-in-time, walk-forward, fee/slippage modeled) plus a sim-trading sandbox that records auditable trade logs plus a public Next.js leaderboard scoring any agent on Sharpe/drawdown/win-rate/expectancy with return decomposition. Ships as a Claude Code skill so other contestants wire their agents in with one line. Evidence: every competitor who tries it = API calls + user count; you seed it with agents run by your Telegram community.

**Concept B: "Vernacular Trader" (Track 1 + moat).** A Telegram-native, multilingual (Pidgin/Swahili/Zulu/English) natural-language trading agent on Agent Hub + GetClaw, with the full perception to decision to execution to risk loop and in-code guardrails, validated by sim trading. Evidence: real African users. Risk: closest to GetClaw, must differentiate on localization + risk safety.

**Concept C: "GuardRail" (Track 2).** A drop-in risk/guardrail middleware for Bitget agents: position limits, leverage caps, drawdown circuit breakers, AgentSpec-style runtime policies, kill-switch, JSON audit trail. Fills a real gap; high adoption potential; safety narrative resonates with judges (and directly answers the BoE's Jonathan Hall "tightly monitored risk and stop-loss limits" concern).

**Concept D: "rStocks Macro Agent" (Track 3).** A macro/sentiment agent for Bitget Stocks 2.0 tokenized equities: reads Fed signals via the `macro-analyst` Skill, auto-rebalances a tokenized-stock sim portfolio, backtests on US-stock historical data, verifiable sim records. Least crowded track; needs custom stock-data plumbing.

**Concept E: "Strategy Compiler" (Track 2).** A natural-language to runnable, backtestable strategy compiler that outputs transparent, auditable strategy code (open analog to the closed Playbook) integrated with Agent Hub execution. Strong DX, reusable.

## The championship pick: reasoning
